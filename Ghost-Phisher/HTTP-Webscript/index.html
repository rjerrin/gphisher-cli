OSVERSION (1003000) do not agree on major version number

February 24, 2017 kalabsd

I was getting the following error on a jail(10.2-RELEASE) running on FreeBSD (12.0-CURRENT) host when trying to install packages from ports.

# make install
make: “/usr/ports/Mk/bsd.port.mk” line 1180: UNAME_r (12.0-CURRENT) and OSVERSION (1003000) do not agree on major version number.

setting UNAME_r environment variable to the jail’s version – 10.2-RELEASE fixed the issue.

# export UNAME_r=10.2-RELEASE

Ansible – backup and upload to aws s3 – FreeBSD

February 21, 2017 kalabsd

Ansible backup and upload to aws s3

This was tested on Freebsd – 10.3 running ansible 2.2.0 against Freebsd 10.3 host 

webbkp and dbbackup ansible roles can be found here

# git clone https://github.com/rjerrin/ansible-roles.git

# cd ansible-roles

# cp -R webbkp /usr/local/etc/ansible/roles

# cp -R  dbbackup  /usr/local/etc/ansible/roles

create a ansible playbook file

#vim backup.yml

– hosts: web
roles:
– webbkp
– dbbackup

vars:
dbname: “mydb”
dbusr: “mydbuser”
passwd: “dbpass”
dbhost: “localhost”
#directory where mysqldump should be saved
bkpdir: “/tmp”
ec2_access_key: “aws-access-key-here”
ec2_secret_key: “aws-secret-key-here”
#directory to be archived and to be uploaded to s3
root_dir: “/usr/local/www”
#name of archive file
bkpfile: “/tmp/www-{{ansible_date_time.date}}.tar.gz”

Run ansible-playbook

# ansible-playbook backup.yml

PLAY [web] *********************************************************************

TASK [setup] *******************************************************************
ok: [omnix.ml]

TASK [webbkp : archive web root] ***********************************************
changed: [omnix.ml]
[WARNING]: Consider using unarchive module rather than running tar
TASK [webbkp : create bucket in s3] ********************************************
ok: [omnix.ml]

TASK [webbkp : upload backup to s3] ********************************************
changed: [omnix.ml]

TASK [webbkp : delete backupfile] *********************************************
changed: [omnix.ml]

TASK [dbbackup : install python mysql module] **********************************
ok: [omnix.ml]

TASK [dbbackup : mysqldump] ****************************************************
changed: [omnix.ml]

TASK [dbbackup : create bucket in s3] ******************************************
ok: [omnix.ml]

TASK [dbbackup : upload backup to s3t] *****************************************
changed: [omnix.ml]

TASK [dbbackup : delete backup file] *******************************************
changed: [omnix.ml]

Kernel Compile – Slacko Puppy Linux

February 21, 2017 kalabsd

Install devx package from   http://puppylinux.org/wikka/devx

Download latest kernel from linux.org

# cd /usr/src
#  wget https://cdn.kernel.org/pub/linux/kernel/v4.x/linux4.10.tar.xz
#  xz -d linux-4.10.tar.xz
#  tar -xvf linux-4.10.tar
# cd linux-4.10
#make clean && make mrproper

Copy config file of currently running kernel

# zcat /proc/config.gz > .config

#make menuconfig

Load currently running kernel configuration, Select LOAD option and enter .config and select ok

make changes to configuration if needed ,select save and then exit

# make
# make modules_install

#cp .config /boot/config-4.10
# cp System.map /boot/System.map-4.10
# cp arch/x86/boot/bzImage /boot/vmlinuz-4.10

Add entry in grub for the new kernel

#vi /menu.lst

#Custom kernel
title Slacko puppy 6.3.2 kernel-4.10
uuid ae54aef2-952d-4044-8c9a-69c111aa9d6a
kernel /boot/vmlinuz-4.10 root=/dev/sdb1 ro

Reboot and choose the new kernel from the boot menu

Ruby – send mail using gmail gem

January 14, 2017 kalabsd

Send mail using ruby gmail gem ( https://github.com/dcparker/ruby-gmail)

Example script :

#!/usr/bin/ruby

require ‘gmail’
EMAILS = Array.new
oauth=”oauthkey”
gmail = Gmail.connect(:xoauth2, ‘user@gmail.com’, oauth)
gmail.inbox.emails( :after => Date.parse(“2017-01-10″), :from => ‘*@domain.com’ ).each do |email|
subject = email.subject
next if subject =~ /walk/i
email.message.body.to_s.scan(/\b[A-Z0-9._%+-]+@[A-Z0-9.-]+\.[A-Z]{2,4}\b/i).each do |x|
next if x =~ /naukri/
next if EMAILS.include?(x)
EMAILS << x
end
subject = subject.gsub(/job|\||urgent|opening|for|resume/i,”)
header = “Resume for” + ” ” + “#{subject}”
puts “#{EMAILS[0]}, #{subject}”
gmail.deliver do
to “#{EMAILS[0]}”
subject “#{header}”
text_part do
body “Please see my attached resume for – #{subject}”
end
add_file “/home/user/Resume.pdf”
end
EMAILS.clear
end
